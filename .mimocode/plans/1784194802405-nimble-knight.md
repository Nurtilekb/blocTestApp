# План: Убрать прямые обращения к CardManager из UI, перевести всё на BLoC

## Проблема

`NoteDetailPage` и `CreateNotePage` напрямую вызывают `CardManager()` для CRUD категорий, обходя BLoC. Это нарушает архитектуру, создаёт дублирование и отсутствие реактивности.

## Что нужно сделать

### Шаг 1: Добавить недостающие методы в `NotesRepository`

**Файл:** `lib/bloc/repositories/notes_repository.dart`

Добавить два метода, которые уже есть в `CardManager`, но не экспортированы в репозиторий:

```dart
void deleteCategory(int id) {
  manager.deleteCategory(id);
}

void updateCategory(int id, String newName) {
  manager.updateCategory(id, newName);
}
```

### Шаг 2: Добавить события категорий в `NotesEvent`

**Файл:** `lib/bloc/notes_event.dart`

Добавить 4 новых sealed-класса:

```dart
class LoadCategories extends NotesEvent {}

class CreateCategory extends NotesEvent {
  final String name;
  const CreateCategory({required this.name});
}

class DeleteCategory extends NotesEvent {
  final int id;
  const DeleteCategory({required this.id});
}

class RenameCategory extends NotesEvent {
  final int id;
  final String newName;
  const RenameCategory({required this.id, required this.newName});
}
```

### Шаг 3: Изменить `NotesState` — хранить `CategoryModel` вместо `List<String>`

**Файл:** `lib/bloc/notes_state.dart`

Изменить тип поля `allCategories` в `NotesLoaded`:

```dart
// Было:
final List<String> allCategories;

// Стало:
final List<CategoryModel> allCategories;
```

Обновить дефолтное значение — генерировать `CategoryModel` из `defaultCategories` с ID 0-3.

### Шаг 4: Добавить обработчики в `NotesBloc`

**Файл:** `lib/bloc/notes_bloc.dart`

1. Зарегистрировать обработчики:
   - `on<LoadCategories>` → `_onLoadCategories`
   - `on<CreateCategory>` → `_onCreateCategory`
   - `on<DeleteCategory>` → `_onDeleteCategory`
   - `on<RenameCategory>` → `_onRenameCategory`

2. Реализовать методы:
   - `_onLoadCategories` — загрузить все категории из `repository.getCategories()`, эмитнуть `NotesLoaded` с обновлённым списком
   - `_onCreateCategory` — `repository.createCategory(name)`, обновить категории в state
   - `_onDeleteCategory` — `repository.deleteCategory(id)`, обновить категории в state
   - `_onRenameCategory` — `repository.updateCategory(id, newName)`, обновить категории в state

3. Убрать `_usedCategories` (Set<String>) — больше не нужен, категории берутся из Hive через репозиторий.

4. В `_onLoadNotes` — загружать и категории вместе с заметками.
