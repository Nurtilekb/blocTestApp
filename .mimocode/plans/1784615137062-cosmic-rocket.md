# Fix: DetailCategoryBadge — пустой список категорий

## Проблема
Когда `allCategories` пуст (категорий нет в Firestore), `_getCategoryId()` крашится на `allCategories.first`, а бейдж показывает некорректное имя старой категории.

## Решение
При пустом списке показывать заглушку **"Добавить категорию"** с иконкой `+`, при нажатии — открывать `CategorySheetContent` для создания первой категории.

## Файл для изменения
`lib/widgets/note_dateail_widgets/detail_category_badge.dart`

## Изменения

### 1. `_getCategoryId()` — добавить проверку на пустой список (уже сделано ранее)
```dart
String _getCategoryId() {
  if (allCategories.isEmpty) return '';
  ...
}
```

### 2. `build()` — разделить на два варианта: пустой и не пустой

Когда `allCategories.isEmpty`:
- Показать тот же стиль бейджа, но с текстом "Добавить категорию" и иконкой `Icons.add`
- При `onTap` (если `isEditing`) — открывать `CategorySheetContent` с пустым списком

Когда `allCategories` не пуст — текущее поведение без изменений.

### 3. Не трогаем
- `note_detail.dart` — там уже есть `isNotEmpty` проверки в `_currentCategoryColor` и `_onCategoryChanged`
- Остальные виджеты

## Верификация
- Запустить приложение, открыть заметку когда нет категорий — должен показаться бейдж "Добавить категорию"
- Нажать на бейдж в режиме редактирования — должна открыться форма создания категории
- После добавления категории — бейдж должен обновиться на выбранную категорию
