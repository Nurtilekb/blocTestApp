# План: Завершение миграции Hive → Firestore

## Статус миграции

**Hive уже полностью удалён из проекта:**
- Все Hive-зависимости в `pubspec.yaml` закомментированы (строки 40-41, 56-57)
- Ни одного импорта Hive нет в `.dart` файлах
- В `pubspec.lock` нет Hive-пакетов
- Сервис Firestore (`lib/services/firestrore_service.dart`) полностью реализован
- `main.dart` инициализирует Firebase и создаёт `FirestoreService`

## Что осталось исправить

### 1. `main.dart` — не используется `DefaultFirebaseOptions`
**Проблема:** `Firebase.initializeApp()` вызывается без `options`, хотя `firebase_options.dart` уже сгенерирован FlutterFire CLI.
**Файл:** `lib/main.dart:16`
**Исправление:** добавить `options: DefaultFirebaseOptions.currentPlatform` и импорт.

### 2. `notes_repository.dart` — методы категорий вложены внутрь `updateNote()`
**Проблема:** строки 26-40 — функции `getCategories()`, `createCategory()`, `getCategoryById()`, `updateNoteCategory()` объявлены как **локальные функции внутри `updateNote()`**. Они недоступны вызывающему коду и не являются методами класса.
**Файл:** `lib/bloc/repositories/notes_repository.dart:23-41`
**Исправление:** вынести их на уровень класса.

### 3. `notes_bloc.dart` — `_onLoadCategories` обращается напрямую к `repository.manager`
**Проблема:** строка 103 — `repository.manager.getAllNotes()` обращается к приватному полю сервиса и вызывает `getAllNotes()` вместо категорий.
**Файл:** `lib/bloc/notes_bloc.dart:99-105`
**Исправление:** использовать `repository.getCategories()` (после исправления п.2).

### 4. `notes_bloc.dart` — пустые обработчики категорий
**Проблема:** `_onCreateCategory` (84-87), `_onDeleteCategory` (89-92), `_onUpdateCategory` (94-97) — пустые тела, ничего не делают.
**Файл:** `lib/bloc/notes_bloc.dart:84-97`
**Исправление:** реализовать вызовы через repository.

### 5. `main.dart` — дублирование `FirestoreService`
**Проблема:** в `main()` создаётся `FirestoreService()` на строке 17, а в `MyApp.build()` создаётся **ещё один** `FirestoreService()` на строке 29. Два разных экземпляра.
**Исправление:** передать один экземпляр через `RepositoryProvider`.

### 6. iOS-конфигурация Firebase отсутствует
**Проблема:** нет файла `ios/Runner/GoogleService-Info.plist`, хотя `firebase_options.dart` содержит iOS-конфиг.
**Исправление:** создать через FlutterFire CLI (`flutterfire configure`) или добавить вручную.

### 7. Пустой файл `card_services.dart`
**Файл:** `lib/services/card_services.dart` — пустой.
**Действие:** удалить, если не планируется использование.

## Порядок исправлений

1. Исправить `notes_repository.dart` — вынести методы категорий на уровень класса
2. Исправить `notes_bloc.dart` — заполнить обработчики категорий через repository
3. Исправить `main.dart` — `DefaultFirebaseOptions` + единый экземпляр сервиса
